# Mizuki 前端交互与体验

本文说明 Mizuki 中页面切换、主题、壁纸、音乐、搜索等前端交互的实现方式及配置方法。

---

## 一、Swup 无刷新页面切换

### 1.1 是什么

Swup 在点击站内链接时，不整页刷新，只替换内容区域，并加上过渡动画，实现类似 SPA 的体验。

### 1.2 工作原理

1. 用户点击链接（如 `/tech/`）
2. Swup 拦截点击，用 `fetch` 请求目标页面
3. 解析返回的 HTML，提取 `#main` 等容器内的内容
4. 用新内容替换当前页面的对应区域
5. 更新浏览器地址栏和 history
6. 执行过渡动画

### 1.3 配置示例（astro.config.mjs）

```js
swup({
  theme: false,
  animationClass: "transition-swup-",
  containers: ["main"],           // 只替换 id="main" 的区域
  cache: true,                    // 缓存已访问页面
  preload: true,                 // 鼠标悬停链接时预加载
  accessibility: true,
  updateHead: true,              // 同步更新 <title>、<meta>
  resolveUrl: (url) => url,
  animateHistoryBrowsing: false,
  skipPopStateHandling: (event) => {
    // 锚点链接交给浏览器原生处理
    return event.state?.url?.includes("#");
  },
})
```

### 1.4 自定义事件

Swup 完成切换后会触发 `mizuki:page:loaded`，其他脚本可监听：

```js
document.addEventListener("mizuki:page:loaded", function () {
  // 新页面加载完成，可在这里初始化组件
});
```

## 二、主题色与暗色模式

### 2.1 主题色（HSL）

`config.ts` 中：

```ts
themeColor: {
  hue: 60,      // 色相 0-360，60 为黄色
  fixed: false, // false 时用户可切换
}
```

- 全站按钮、链接、高亮等使用 `--primary`，由 `hue` 计算
- `fixed: true` 时隐藏主题色选择器，用户无法切换

### 2.2 暗色模式

- 通过 `prefers-color-scheme` 或用户手动切换
- 样式用 CSS 变量区分亮/暗，如 `--content-90`、`--btn-regular-bg` 等

## 三、壁纸模式

### 3.1 三种模式

| 模式 | 说明 |
|------|------|
| `banner` | 顶部横幅，占据顶部区域 |
| `fullscreen` | 全屏壁纸，内容浮在上方 |
| `none` | 无壁纸 |

### 3.2 配置示例

```ts
wallpaperMode: {
  defaultMode: "banner",
  showModeSwitchOnMobile: "desktop",  // 移动端不显示切换按钮
},

banner: {
  src: {
    desktop: ["/assets/web.jpg"],
    mobile: ["/assets/web.jpg"],
  },
  waves: {
    enable: true,   // 水波纹效果
    performanceMode: false,
  },
}
```

### 3.3 水波纹效果

`waves.enable: true` 时，横幅区域会有动态水波纹，对性能有一定影响，可设 `performanceMode: true` 降低复杂度。

## 四、音乐播放器

### 4.1 三种模式

| 模式 | 说明 |
|------|------|
| `meting` | 网易云等，通过 Meting API 拉歌单 |
| `local` | 本地 MP3，需把文件放在 `public/assets/music/` |
| `spotify` | Spotify 歌单嵌入 |

### 4.2 Meting 模式配置

```ts
musicPlayerConfig: {
  enable: true,
  mode: "meting",
  meting_api: "https://meting.xxx/api?server=:server&type=:type&id=:id&auth=:auth&r=:r",
  id: "14164869977",    // 网易云歌单 ID
  server: "netease",    // netease | tencent | kugou 等
  type: "playlist",
}
```

### 4.3 Spotify 模式配置

```ts
musicPlayerConfig: {
  enable: true,
  mode: "spotify",
  spotifyPlaylistId: "0SAg9BnVnUGj88OmYqUYIO",  // 从歌单链接获取
}
```

歌单链接格式：`https://open.spotify.com/playlist/xxx`，其中 `xxx` 即为 ID。

### 4.4 播放器位置

- 右下角浮动
- 可收缩为小圆球、展开为完整播放器
- 支持隐藏（点击眼睛图标）

## 五、Pagefind 站内搜索

### 5.1 构建时

`pnpm build` 会执行 `pagefind --site dist`，扫描 `dist/` 下的 HTML，生成索引到 `dist/pagefind/`。

### 5.2 前端使用

- 页面引入 `pagefind.js` 和 `pagefind-ui.js`
- 用户输入关键词后，在索引中检索并展示结果
- 支持中文，但无词干提取（stemming）

### 5.3 排除某些内容

给不想被索引的元素加 `data-pagefind-ignore`：

```html
<nav data-pagefind-ignore>...</nav>
```

## 六、相册与图片展示

### 6.1 相册页面

- 数据来自 `src/data/` 或 `public/albums/` 下的配置
- 使用 PhotoSwipe 实现点击放大、左右切换

### 6.2 文章内图片

- 用 Markdown 语法：`![alt](url)`
- 点击可放大（若启用了相关组件）

### 6.3 Fancybox

部分灯箱效果使用 Fancybox，用于弹窗展示图片或链接内容。

---

## 七、其他交互

### 7.1 目录（TOC）

- 文章页根据标题自动生成目录
- 可配置 `toc.enable` 开关

### 7.2 分享

- 支持生成分享图、复制链接等
- 由 `shareConfig` 控制

### 7.3 评论

- 可集成 Twikoo、Waline 等
- 需在 `config.ts` 中配置并启用
