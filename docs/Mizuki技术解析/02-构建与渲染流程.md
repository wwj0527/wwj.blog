# Mizuki 构建与渲染流程

本文按步骤拆解 `pnpm build` 做了什么，以及从源码到最终静态 HTML 的完整流程。

---

## 一、构建命令解析

### 1.1 完整构建命令

```bash
pnpm build
```

实际执行的是 `package.json` 里定义的：

```json
"build": "node scripts/update-anime.mjs && astro build && pagefind --site dist && node scripts/compress-fonts.js"
```

即按顺序执行 4 步：

| 步骤 | 命令 | 作用 |
|------|------|------|
| 1 | `node scripts/update-anime.mjs` | 更新番剧数据（若启用 Bangumi/B 站） |
| 2 | `astro build` | Astro 构建，生成 `dist/` |
| 3 | `pagefind --site dist` | 扫描 dist，生成站内搜索索引 |
| 4 | `node scripts/compress-fonts.js` | 压缩字体（若启用） |

### 1.2 每一步详解

**步骤 1：update-anime.mjs**

- 若 `anime.mode` 为 `bangumi` 或 `bilibili`，会请求 API 拉取番剧数据
- 结果写入 `public/` 或 `src/data/` 下的 JSON
- 若为 `local`，通常直接跳过

**步骤 2：astro build**

- 编译所有 `.astro`、`.svelte`、`.ts` 页面
- 解析 `src/content/` 下的 Markdown
- 输出到 `dist/`，每个页面对应一个 HTML 文件

**步骤 3：pagefind**

- 扫描 `dist/` 下的 HTML
- 提取正文、标题等，生成索引到 `dist/pagefind/`
- 前端搜索时加载这些索引做全文检索

**步骤 4：compress-fonts.js**

- 若配置了字体压缩，会处理 `public/` 下的字体文件
- 默认可能关闭（`enableCompress: false`）

## 二、Astro 构建内部流程

### 2.1 页面收集

Astro 会扫描 `src/pages/`，根据文件名生成路由：

| 文件路径 | 生成路由 | 说明 |
|----------|----------|------|
| `[...page].astro` | `/`、`/2/`、`/3/`... | 首页及分页 |
| `about.astro` | `/about/` | 关于页 |
| `archive.astro` | `/archive/` | 归档 |
| `tech.astro` | `/tech/` | 技术分类 |
| `posts/[...slug].astro` | `/posts/xxx/` | 文章详情（动态路由） |
| `rss.xml.ts` | `/rss.xml` | RSS |
| `404.astro` | 404 页面 | 未找到时显示 |

### 2.2 动态路由示例：文章页

`posts/[...slug].astro` 会根据 `src/content/posts/` 下的 Markdown 生成多页：

```
src/content/posts/
  ├── git-guide.md          → /posts/git-guide/
  ├── markdown-guide.md     → /posts/markdown-guide/
  └── guide/
      └── index.md          → /posts/guide/
```

构建时 Astro 会：

1. 读取 content 配置，获取所有文章
2. 对每篇文章调用一次页面组件，传入 `slug` 等参数
3. 生成对应的 `dist/posts/xxx/index.html`

### 2.3 构建输出目录结构

```
dist/
├── index.html              # 首页
├── about/
│   └── index.html
├── tech/
│   └── index.html
├── posts/
│   ├── git-guide/
│   │   └── index.html
│   └── ...
├── _astro/                 # JS、CSS 等静态资源
│   ├── xxx.js
│   ├── xxx.css
│   └── ...
├── pagefind/               # 搜索索引（pagefind 生成）
│   ├── pagefind.js
│   └── ...
├── rss.xml
├── sitemap-index.xml
└── assets/                 # 从 public/ 复制过来的资源
```

## 三、资源处理

### 3.1 图片

- 放在 `public/` 的图片会原样复制到 `dist/`
- 在组件里用 `import` 引入的图片，Astro 会处理路径并参与打包

**示例**：

```astro
---
import logo from "../assets/images/logo.png";
---
<img src={logo.src} alt="Logo" />
```

### 3.2 字体

- `@fontsource` 等 npm 包提供的字体会被打包进 `_astro/`
- `public/` 下的字体直接复制，可用 `compress-fonts` 做压缩

### 3.3 JS 分包

- Astro 会按页面和组件做 code splitting
- 每个页面只加载自己用到的 JS，减小首屏体积

## 四、开发模式 vs 构建模式

### 4.1 开发模式

```bash
pnpm dev
```

- 实际执行 `astro dev`（`predev` 会先跑 `sync-content`）
- 启动本地服务器，默认 `http://localhost:4321`
- 文件改动会热更新，不生成 `dist/`

### 4.2 预览构建结果

```bash
pnpm build
pnpm preview
```

- `preview` 会启动一个静态服务器，直接托管 `dist/`
- 用于在本地验证构建结果，接近线上环境

## 六、路由与 trailingSlash

`astro.config.mjs` 中有：

```js
trailingSlash: "always"
```

表示所有 URL 都以 `/` 结尾，例如：

- `/about` → `/about/`
- `/posts/git-guide` → `/posts/git-guide/`

因此 `dist/` 下每个页面都是 `index.html` 放在对应目录中，便于服务器配置。
