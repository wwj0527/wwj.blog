# Mizuki 技术栈总览

> 原作者：matsuzaka-yuki | [GitHub](https://github.com/matsuzaka-yuki/Mizuki) | [演示站](https://mizuki.mysqil.com/)

本文详细梳理 Mizuki 主题用到的框架、库和工具，并说明各自的作用和典型用法。

---

## 一、核心框架

### 1.1 Astro

**是什么**：以内容为核心的静态站点生成框架，默认输出纯 HTML，按需加载 JS。

**在 Mizuki 中的角色**：
- 负责页面路由、布局、Markdown 解析
- 构建时把 `.astro`、`.svelte` 等编译成静态 HTML
- 配置在 `astro.config.mjs`

**示例：页面如何定义**

```astro
<!-- src/pages/tech.astro -->
---
import MainGridLayout from "../layouts/MainGridLayout.astro";
import { siteConfig } from "../config";
---

<MainGridLayout title="编程与技术" description="技术类文章">
  <p>这里是技术分类页面</p>
</MainGridLayout>
```

- 文件放在 `src/pages/` 下，路径即路由：`tech.astro` → `/tech/`
- `---` 之间是脚本，下面是模板

### 1.2 Svelte

**是什么**：编译型前端框架，把组件编译成原生 DOM 操作，运行时体积小。

**在 Mizuki 中的角色**：
- 需要交互的组件用 Svelte 写，如音乐播放器、搜索、主题切换
- 通过 `@astrojs/svelte` 集成

**示例：Svelte 组件结构**

```svelte
<!-- 简化示例：音乐播放器的一部分 -->
<script lang="ts">
  let isPlaying = false;
  function togglePlay() {
    isPlaying = !isPlaying;
  }
</script>

<button on:click={togglePlay}>
  {isPlaying ? "暂停" : "播放"}
</button>
```

- `script` 里写逻辑，模板里用 `{变量}` 和 `on:click` 等
- 构建时会被编译成普通 JS，无需在页面加载完整框架

## 二、样式体系

### 2.1 Tailwind CSS

**是什么**：工具类 CSS 框架，用类名组合实现样式。

**在 Mizuki 中的用法**：

```html
<!-- 常见类名示例 -->
<div class="flex items-center gap-3">        <!-- 弹性布局、垂直居中、间距 -->
<div class="text-90 text-sm truncate">      <!-- 文字颜色、字号、超出省略 -->
<div class="rounded-2xl shadow-xl">         <!-- 圆角、阴影 -->
<div class="bg-(--float-panel-bg)">         <!-- 使用 CSS 变量 -->
```

- `flex`、`gap-3` 等是 Tailwind 内置
- `--float-panel-bg` 是主题里定义的 CSS 变量，用 `bg-(--变量名)` 引用

### 2.2 Stylus

**是什么**：CSS 预处理器，支持变量、嵌套、混合。

**在 Mizuki 中的用法**：

```stylus
// src/styles/variables.styl 中定义主题变量
$primary-hue = 60
$content-90 = hsl(var(--content-90-hsl))
```

- 主要用于变量和复杂样式
- 编译后生成普通 CSS

### 2.3 CSS 变量（主题色）

**是什么**：用 `--变量名` 定义可动态修改的样式值。

**示例：主题色切换**

```css
/* 主题色由 config.ts 的 themeColor.hue 控制 */
:root {
  --primary: hsl(60, 70%, 50%);   /* 色相 60 = 黄色 */
}

/* 用户切换主题色时，只改 --primary，全站按钮、链接等自动更新 */
```

## 三、内容处理：Markdown 与插件

### 3.1 remark（Markdown → 抽象语法树）

**流程**：Markdown 文本 → remark 解析 → 抽象语法树（AST）

**Mizuki 使用的 remark 插件**：

| 插件 | 作用 | 示例 |
|------|------|------|
| `remarkMath` | 数学公式 | `$E=mc^2$` → 公式节点 |
| `remarkMermaid` | Mermaid 图表 | ` ```mermaid ... ``` ` → 图表 |
| `remarkDirective` | 自定义指令 | `:::note 提示 :::` → 提示框 |
| `remarkSectionize` | 章节包裹 | 给标题和内容分组 |

### 3.2 rehype（AST → HTML）

**流程**：AST → rehype 转换 → HTML 字符串

**Mizuki 使用的 rehype 插件**：

| 插件 | 作用 | 示例 |
|------|------|------|
| `rehypeKatex` | 数学公式渲染 | 公式节点 → `<span class="katex">` |
| `rehypeMermaid` | Mermaid 渲染 | 图表节点 → `<div class="mermaid">` |
| `rehypeSlug` | 标题 id | `## 标题` → `<h2 id="标题">` |
| `rehypeAutolinkHeadings` | 标题锚点 | 给标题加可点击的 `#` 链接 |
| `rehypeComponents` | 自定义 HTML 组件 | `note`、`tip`、`github` 等 |

## 四、功能库一览

### 4.1 页面切换：Swup

**是什么**：无刷新页面切换库，点击链接时只替换内容区，不整页重载。

**效果**：切换页面时没有白屏，过渡更顺滑。

**配置示例**（`astro.config.mjs`）：

```js
swup({
  containers: ["main"],      // 只替换 id="main" 的区域
  cache: true,             // 缓存已访问页面
  preload: true,           // 鼠标悬停链接时预加载
})
```

### 4.2 代码高亮：Expressive Code

**是什么**：代码块渲染库，支持行号、折叠、主题等。

**Markdown 中的用法**：

````markdown
```js
function hello() {
  console.log("Hello");
}
```
````

- 自动加行号、语法高亮
- 支持 `github-light`、`github-dark` 主题

### 4.3 站内搜索：Pagefind

**是什么**：构建时生成搜索索引，前端用 JS 做全文搜索。

**构建流程**：
1. `pnpm build` 生成 `dist/`
2. `pagefind --site dist` 扫描 HTML，生成索引到 `dist/pagefind/`
3. 页面引入 `pagefind.js`，用户输入关键词即可搜索

### 4.4 图片展示

| 库 | 用途 | 场景 |
|------|------|------|
| **PhotoSwipe** | 相册、大图预览 | 相册页、文章内图片点击放大 |
| **Fancybox** | 灯箱效果 | 部分弹窗展示 |
| **Sharp** | 图片处理 | 构建时压缩、生成 WebP |

### 4.5 其他

| 库 | 用途 |
|------|------|
| **KaTeX** | 数学公式渲染（比 MathJax 轻量） |
| **dayjs** | 日期格式化 |
| **axios** | HTTP 请求（Bangumi、B 站等 API） |
| **crypto-js** | 加密（加密文章等） |
| **satori** | 生成 OG 图片（社交分享预览图） |
| **oddmisc** | Umami 统计集成 |

## 五、依赖关系简图

```
Astro (核心)
  ├── @astrojs/svelte     → Svelte 组件
  ├── @astrojs/sitemap    → 站点地图
  ├── @astrojs/rss        → RSS 订阅
  ├── astro-expressive-code → 代码块
  ├── swup                → 页面切换
  └── remark/rehype 插件  → Markdown 处理

Tailwind CSS
  └── @tailwindcss/vite   → Vite 插件形式集成

构建后
  └── pagefind            → 扫描 dist 生成搜索索引
```

## 六、如何查看项目用了哪些库

```bash
# 查看所有依赖
pnpm list

# 查看顶层依赖（不含子依赖）
pnpm list --depth 0
```

`package.json` 的 `dependencies` 和 `devDependencies` 里列出了全部依赖及版本。
